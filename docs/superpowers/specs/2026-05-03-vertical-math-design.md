# 乘除法竖式练习 - 设计文档

## 概述

面向 6-8 岁小学生的整数乘法和可整除除法竖式计算展示工具。单页面应用，随机出题，点击"解答"后以彩色竖式展示完整计算过程。

## 需求

- 出题范围：乘法乘数 ≤100（最大 100×100），除法除数 ≤100（如 10000÷100）
- 交互：出题后不立即给答案，点击"解答"后显示竖式结果
- 模式：纯展示式，不要求孩子填写答案
- 出题方式：单题随机，乘除法完全随机

## 架构

单页面应用，所有功能在一个页面完成。

```
app/
├── layout.tsx              # 根布局
├── page.tsx                # 主页面（出题 + 竖式展示 + 按钮）
├── globals.css             # 全局样式 + CSS 变量
└── components/
    ├── MultiplicationVertical.tsx   # 乘法竖式渲染组件
    └── DivisionVertical.tsx         # 除法竖式渲染组件
```

## 页面布局

移动端优先，居中显示，从上到下四个区域：

1. **标题栏**：紫色渐变背景，显示"乘除法竖式练习"
2. **题目区**：灰底，大号字体显示题目（如 `23 × 15 = ?`），解答后显示等号和结果
3. **竖式区**：白色背景，初始提示"点击解答按钮查看竖式过程"，解答后展示彩色竖式
4. **按钮区**：两个大按钮 — 绿色「出题」、蓝色「解答」

## 竖式视觉设计

### 通用原则

- 使用 CSS Grid 实现精确的列对齐
- 每个数字独占一个色块（圆角背景 + 深色粗体字）
- 运算符号（×、+、−）统一红色 `#C62828`，紧贴对应数字左侧

### 颜色体系

| 颜色 | 背景色 | 文字色 | 用途 |
|------|--------|--------|------|
| 绿色 | `#A5D6A7` | `#1B5E20` | 原始数：被乘数、被除数、落下来的位、余数 |
| 蓝色 | `#90CAF9` | `#0D47A1` | 运算数：乘数、除数 |
| 橙色 | `#FFCC80` | `#E65100` | 中间过程：部分积、减去的数、差值 |
| 红色 | `#EF9A9A` | `#B71C1C` | 最终结果：积、商 |
| 符号 | 无背景 | `#C62828` | 运算符号：×、+、− |

### 乘法竖式布局

以 23 × 15 = 345 为例：

```
Grid 列: [符号 24px] [数位 36px] [数位 36px] [数位 36px]

     [ ]    [ ]    [2]绿  [3]绿        ← 被乘数
     [ ]    [×]红  [1]蓝  [5]蓝        ← 乘数
     [────────────────────────]        ← 横线
     [ ]    [1]橙  [1]橙  [5]橙        ← 部分积₁ (×5)
     [+]红  [2]橙  [3]橙  [0]橙        ← 部分积₂ (×10)
     [────────────────────────]        ← 横线
     [ ]    [3]红   [4]红  [5]红        ← 最终结果
```

- 符号列固定宽度，× 和 + 分别紧贴各自数字左列
- 所有数位右对齐，保证位值对应

### 除法竖式布局

以 345 ÷ 15 = 23 为例：

```
外层: [除数] [除号区域]
               商: [2]红 [3]红        ← 在横线上方
除号区域: border-left + border-top 构成 L 形

Grid 列: [符号 20px] [数位 36px] [数位 36px] [数位 36px]

[ ]  [3]绿  [4]绿  [5]绿              ← 被除数
[−]红 [3]橙  [0]橙                    ← 减: 15×2=30
[────────────────────]                ← 横线
[ ]   [ ]    [4]橙  [5]绿              ← 差+落位（5 从被除数落下，保持绿色）
[ ]   [−]红  [4]橙  [5]橙              ← 减: 15×3=45
[ ]   [────────────────]              ← 横线
[ ]   [ ]    [ ]    [0]绿              ← 余数
```

- 除号用 CSS border-left + border-top 构成完整 L 形
- 商位于横线上方，与被除数右对齐
- 减号（−）紧贴对应数字左侧列
- 从被除数落下来的位保持绿色，与原始数区分
- 符号随数字位置移动（如第二个 − 在 col2 而非 col1）

## 数据流

1. **出题**：随机选择乘法或除法 → 在指定范围内生成合法题目 → 更新题目状态
2. **解答**：根据题目类型执行竖式算法 → 生成每一步的数据 → 渲染竖式组件

### 题目生成规则

- 50% 概率生成乘法或除法
- 乘法：两个乘数均 1-100 随机
- 除法：先生成除数（1-100）和商（1-100），再计算被除数，保证整除

### 竖式计算数据结构

乘法步骤：
```
{ multiplicand, multiplier, partialProducts: number[], product }
```

除法步骤：
```
{ dividend, divisor, steps: [{ quotientDigit, subtractValue, remainder, broughtDown? }], quotient }
```
